## 13.0 OPC开发哲学

> **开发模式**：一人公司（OPC），一名全栈工程师负责全部开发。以模块为单位串行开发，确保一个模块完成并验证后再启动下一个。每个模块内部遵循「前端先行→后端开发→联调」的节奏推进。
>
> **总工期估算**：~226工作日（含15%缓冲约260天 ≈ 52周 ≈ 12个月）
>
> **MVP范围**：全部11个模块（M0基建 + M1-M10业务模块），不砍功能。

### 13.0.1 核心原则

- **串行不并行**：一次只做一个模块，降低上下文切换成本
- **前端先行**：每个模块先完成前端页面+Mock数据，验证交互后再开发后端
- **独立可验证**：每个模块有明确的验收标准，可独立演示
- **依赖最小化**：模块间通过API接口解耦，避免跨模块耦合
- **每日可交付**：每天结束时代码可编译、可运行，不留下半成品
- **AI辅助提效**：善用Cursor/Copilot等AI编程工具，目标提升30%编码效率

### 13.0.3 一人开发缓冲策略

| 策略 | 说明 |
|------|------|
| **15%时间缓冲** | 每个模块工期上浮15%，应对不可预见问题 |
| **周五Review日** | 每周五下午做模块Review，检查进度偏差 |
| **技术债务日** | 每个模块联调阶段预留1天处理累积技术债 |
| **风险前置** | 高风险模块（M2简历解析、M4智能匹配）优先开发，趁精力充沛时攻克 |
| **周末不加班** | 严格执行5天工作制，避免burnout |
| **里程碑Demo** | 每个模块完成后录制5分钟Demo视频，作为交付确认 |
