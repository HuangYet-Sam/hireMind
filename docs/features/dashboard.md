# 工作台Dashboard + AI任务中心

> **PRD来源**：§4.0a 工作台页面Dashboard（L913-L962）+ §4.0b AI任务中心
> **PRD版本**：V3.6 | **最后更新**：2026-05-12

---

## 功能概述

**路由**：`/hr/dashboard` | **新增于**：第十二轮研讨 | **优先级**：P0

HR打开系统的第一个页面——**行动优先级驱动**的工作台。HR每天的第一件事是"今天我要做什么"——哪些面试、哪些简历没看、哪些Offer要跟、哪些岗位超期了。

### 里程碑策略
- **M1-M2交付**：核心指标+待办（紧急待办区+今日日程+快捷指标），**不含AI洞察**
- **M3交付**：AI洞察推送区域（基于主动式AI引擎）

---

## 页面布局

### 1. 🔴 紧急待办区
- 简历待筛选
- 面试待反馈
- Offer待跟进
- 审批待处理
- 岗位超期未招到

每项附「去处理→」快捷链接。

### 2. 📅 今日日程
- 当日面试排期（时间+候选人+岗位+面试官+状态）
- 内部会议

### 3. 📊 本周快捷指标
- 新增简历（含周同比）
- 面试通过率
- 平均到岗天数
- 匹配命中率
- Offer接受率
- 活跃岗位数

### 4. 🤖 AI洞察（M3交付，主动推送）
- 高匹配候选人推荐
- 岗位超期建议
- 流程瓶颈预警

---

## 支撑端点

| 端点 | 用途 |
|------|------|
| `GET /api/v1/dashboard/todos` | 紧急待办列表 |
| `GET /api/v1/dashboard/schedule` | 今日面试日程 |
| `GET /api/v1/dashboard/metrics` | 本周快捷指标 |
| `GET /api/v1/dashboard/ai-insights` | AI主动洞察推送 |

---

## 数据来源

- **待办数据**：实时查询各业务模块（简历状态/面试状态/Offer状态/审批流/岗位超期）
- **日程数据**：面试模块当日排期
- **指标数据**：定时聚合（Cron每日凌晨）+ 实时增量更新
- **AI洞察**：Cron每日凌晨生成+缓存，HR可手动刷新
