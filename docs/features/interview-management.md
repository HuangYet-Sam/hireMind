## 功能概述

**功能ID**：F5 | **优先级**：P0 | **角色**：HR、面试官 | **映射**：US-015~018

面试模块从"排期+反馈收集"升级为**全链路AI面试助手**——面试前AI生成考察清单+验证要点，面试中录音/视频实时分析，面试后AI出具多维度分析报告，全面辅助HR和用人部门进行面试评估与决策。

### 嵌入式面试（不走本页面，已嵌入360°视图）
- 岗位360°视图 → Tab4面试安排 → 查看该岗位所有面试
- 候选人360°视图 → Tab3面试记录 → 查看该候选人所有面试历程

## 面试流程模板

每个岗位可配置面试轮次流程模板，定义面试阶段数、每轮形式和评估维度。

| 模板字段 | 类型 | 说明 |
|---------|------|------|
| `interview_rounds` | JSON | 面试轮次配置数组，存储在`positions.interview_config` |
| `rounds[].round` | INT | 轮次序号 |
| `rounds[].name` | STRING | 轮次名称 |
| `rounds[].format` | ENUM | 面试形式：`onsite`/`video`/`phone` |
| `rounds[].duration_min` | INT | 建议时长（分钟） |
| `rounds[].eval_dimensions` | STRING[] | 评估维度 |
| `rounds[].interviewer_role` | STRING | 建议面试官角色 |

### 默认模板
```
R1: 初筛（电话/视频, 30min, HR）
R2: 技术面（视频/现场, 60min, 用人部门）
R3: 终面（现场, 45min, 部门负责人/HRBP）
```

## AI面试考察清单

每场面试前AI自动生成**面试官Briefing**，包含4类考察项：

| 类型 | 说明 | 时间占比 |
|------|------|---------|
| 🔴 必考项 | 岗位核心要求，画像中缺失或弱项 | 40% |
| 🟡 验证项 | 简历可信度检测发现的问题 | 30% |
| 🟢 确认项 | 候选人声称的强项，快速确认 | 10% |
| 🧠 隐性探测 | AI推理出的潜在能力 | 20% |

---

## AI面试分析报告

面试结束后自动生成，包含：
1. **综合评估**：评分（技术深度/项目实战/系统设计/沟通表达/问题解决）
2. **考察清单完成情况**：每项通过/未通过+佐证描述
3. **关键发现**：优势+风险点+AI录用建议+薪资建议+试用期关注点
4. **画像更新预览**：本场面试将更新候选人画像的内容，需HR确认

## 业务规则

- 面试双FK：必须同时关联`position_id`和`candidate_id`
- 多轮次：`round` + `stage`
- 考察清单自动生成：面试创建后自动触发
- 录音/视频：可选，需候选人授权
- AI实时辅助延迟<3秒
- 画像更新闭环：报告中每条"更新画像"项需HR确认后才写入
- Token冲突规则：Token提交优先 → HR代填降为草稿
- P2-25面试官回避机制（Phase 2实现）

---

## 面试官回避机制（Phase 2）

- 面试安排时增加利益冲突检查
- Phase 1：HR手动确认无利益冲突
- Phase 2：接入HRIS数据，自动检测
- 数据模型预留：`interviews.conflict_check_status ENUM('pending/confirmed/flagged')`
