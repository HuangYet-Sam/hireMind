# 招聘分析

> **PRD来源**：§4.7 招聘分析（L2681-L2787）
> **PRD版本**：V3.6 | **最后更新**：2026-05-12

---

## 功能概述

**功能ID**：F7 | **优先级**：P0 | **角色**：HR、用人经理（只读Dashboard） | **映射**：US-022~024

招聘分析看板以全局仪表盘为入口，支持按岗位/部门/渠道钻取，AI洞察直接给出行动建议而非数据罗列。分析不是静态图表的堆砌，而是可交互、可钻取、可行动的决策工具。

## 全局仪表盘

路由：`/hr/analytics`

### 页面结构
1. **时间范围+部门筛选器**
2. **KPI卡片行**：在招岗位数/候选人总数/本周面试/待审Offer
3. **中部图表区（双列）**：
   - 招聘漏斗（全局）：收简历→已匹配→已面试→已Offer→已入职
   - 岗位健康度：🔴空缺 🟡部分空缺 🟢已满
4. **底部区域**：
   - AI洞察（趋势+行动建议）
   - 渠道ROI（面试率+成本）

## 漏斗钻取

点击漏斗任一阶段→展开该阶段详情：
- 通过率趋势（月度）
- 失败维度Top3
- AI建议（如"调整JD中系统设计要求"）

### 钻取级别
全局 → 部门 → 岗位 → 单个阶段

---

## 业务规则

- 仪表盘数据实时更新（SSE推送或定时轮询30秒）
- 漏斗支持三级钻取
- AI洞察由Cron每日凌晨生成+缓存，HR可手动刷新
- 岗位健康度算法：空缺比例×持续天数×简历流入速率
- 渠道ROI = 渠道总投入 / 该渠道产生的面试数量
- 周报/月报通过Cron自动生成，邮件+IM双通道推送

### 招聘漏斗"已入职"计算逻辑
- 漏斗"已入职"= `matches`表中`status=hired`且`offers`表`onboarded_at IS NOT NULL`
- 漏斗各阶段定义：收简历→已匹配(`shortlisted`+)→已面试(`interviewing`+)→已发Offer(`offered`+)→已入职(`onboarded`)
