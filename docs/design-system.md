> 版本: v1.0 | 最后更新: 2026-05-12 | 作者: Designer
>
> ⚠️ **本文档基于 Next.js + React + Prisma 架构编写，技术栈已变更为 Vue 3 + FastAPI + SQLAlchemy。** 文档中的业务设计、数据模型、API接口定义仍然有效，但所有技术实现细节（代码示例、组件引用、配置方式）需要按新架构重新理解。具体技术差异参见 `docs/architecture/tech-stack.md`。

## 1. 概述

### 1.1 文档目的

本设计系统规范定义 HireMind 的视觉语言、组件使用约定和交互模式。所有 UI 开发必须以此为基准，确保产品视觉一致性和用户体验质量。

### 1.2 设计哲学来源

本设计系统遵循 HireMind 五条产品哲学：

1. **零侵入原则** — AI 功能用温和视觉提示，不弹窗不遮挡
2. **论文式画像** — 每条数据旁有引用标记，悬停显示出处，点击跳转
3. **AI 隐性推理** — AI 推理结果有独特视觉标记，与事实数据明确区分
4. **简洁实用** — 信息密度适中，减少点击和页面跳转
5. **360度全息视图** — 分区块布局，每个区块聚合一类上下文

### 1.3 技术基础

| 技术 | 版本 | 用途 |
|------|------|------|
| Tailwind CSS | 4.x | 原子化样式 |
| shadcn/ui | 最新 | 基础组件库 |
| Lucide React | 最新 | 图标库 |
| CSS Variables | - | 主题切换机制 |

### 1.4 设计原则

| 原则 | 核心思想 | 应用方式 |
|------|----------|----------|
| 认知负荷 | Miller's Law (7±2) | 单屏信息分组不超过 7 块 |
| 渐进披露 | 核心优先，详情按需 | 默认折叠，展开查看 |
| 菲茨定律 | 高频操作大且可达 | 保存/发送按钮突出，删除远离 |
| 格式塔原则 | 相近相似 | 同类信息紧凑排列，相同样式 |

### 2.5 色彩使用规则

| 场景 | 色彩 | 说明 |
|------|------|------|
| 品牌蓝 (primary) | 主按钮、导航活跃、关键操作 | 每屏最多出现 2-3 处 |
| 语义反馈 | 成功/警告/危险 | 仅用于状态反馈，不做装饰 |
| AI 专属色 | AI 推理结果、AI 标记 | 必须与事实数据有明确视觉区分 |
| 中性色 | 文字、边框、背景 | 占页面 80% 以上面积 |

**禁忌：**
- 不用 AI 紫色做主操作按钮
- 不用品牌蓝标注 AI 内容
- 不在同一页面使用超过 3 种语义反馈色
- 不使用纯黑 (#000) 和纯白 (#FFF) 以外的极端对比

## 3. 排版规范

### 3.1 字体栈

```
--font-sans: 'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;
```

Inter 作为主字体，覆盖中文回退到系统字体。等宽字体用于代码片段和数据展示。

### 3.2 字号层级

遵循三级信息层次：标题 > 正文 > 辅助。

| 层级 | Token | Tailwind | 用途 |
|------|-------|----------|------|
| H1 | --text-h1 | text-2xl (24px) | 页面标题 |
| H2 | --text-h2 | text-xl (20px) | 区块标题 |
| H3 | --text-h3 | text-lg (18px) | 卡片标题、子区块 |
| H4 | --text-h4 | text-base (16px) | 列表项标题 |
| Body | --text-body | text-sm (14px) | 正文、表单标签、描述 |
| Caption | --text-caption | text-xs (12px) | 辅助文字、时间戳、引用标记 |
| Micro | --text-micro | text-[11px] | Badge 内文字、极度紧凑场景 |

### 3.3 字重

| Token | Tailwind | 用途 |
|-------|----------|------|
| --font-normal | font-normal (400) | 正文、描述 |
| --font-medium | font-medium (500) | 标签、表头、小标题 |
| --font-semibold | font-semibold (600) | 页面标题、区块标题 |
| --font-bold | font-bold (700) | 极少使用，仅数据高亮 |

### 3.4 行高

| 场景 | 行高 | Tailwind |
|------|------|----------|
| 标题 | 1.3 | leading-tight |
| 正文 | 1.5 | leading-normal |
| 紧凑文字 | 1.25 | leading-snug |
| 宽松文字 | 1.75 | leading-relaxed |

### 3.5 排版约束

- 单行文字不超过 80 字符（中文约 40 字），超出换行
- 段落间距 16px (space-y-4)
- 标题与正文间距 8px (space-y-2)
- 不使用全大写英文（UPPERCASE），仅 Badge 允许
- 不使用斜体，用颜色或粗细区分重点

| --space-3 | 12px | 卡片内间距（紧凑） |
| --space-4 | 16px | 卡片内间距（标准）、段落间距 |
| --space-5 | 20px | 区块内子元素间距 |
| --space-6 | 24px | 区块间距（紧凑） |

### 4.2 页面栅格

```
页面结构：
┌──────────────────────────────────────────────────┐
│  Sidebar (w-64 / 256px, 固定)                      │
│  ┌──────────────────────────────────────────────┐ │
│  │  Header (h-14 / 56px, 固定)                   │ │
│  ├──────────────────────────────────────────────┤ │
│  │  Main Content (max-w-7xl / 1280px 居中)       │ │
│  │  padding: px-6 py-8                           │ │
│  │                                               │ │
│  │  内部可用 12 列栅格，gutter 24px               │ │
│  └──────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

**关键尺寸：**

| 元素 | 尺寸 | 说明 |
|------|------|------|
| 侧边栏 | 256px (w-64) | 可折叠至 64px (w-16, 仅图标) |
| 顶栏 | 56px (h-14) | 面包屑 + 全局操作 |
| 内容最大宽度 | 1280px (max-w-7xl) | 超宽屏居中 |
| 内容内边距 | px-6 py-8 | 左右 24px，上下 32px |
| 栅格间距 | gap-6 (24px) | 多列布局的列间距 |

### 4.3 响应式断点

| 断点 | 宽度 | 适配 |
|------|------|------|
| Desktop | >= 1280px | 完整侧边栏 + 多列 |
| Tablet | 768-1279px | 折叠侧边栏 + 双列 |
| Mobile | < 768px | 不做主动适配（HR 桌面端优先） |

MVP 阶段以 Desktop 为主要适配目标，Tablet 基本可用即可，Mobile 不主动适配。

## 6. shadcn/ui 组件使用约定

### 6.1 必用组件

以下 shadcn/ui 组件作为项目基础组件，必须通过 `npx shadcn-ui@latest add` 安装到 `components/ui/`：

| 组件 | 用途 | 定制要求 |
|------|------|----------|
| Button | 所有按钮 | 4 种 variant: default/secondary/outline/ghost |
| Input | 文本输入 | 统一高度 h-9，与 Search/Input 一致 |
| Select | 下拉选择 | 宽度自适应或 w-full |
| Textarea | 多行输入 | 最小高度 min-h-[80px] |
| Dialog | 模态框 | 仅用于确认删除等危险操作，AI 功能不用 |
| Sheet | 侧边面板 | 候选人详情快速查看（非完整详情页） |
| Card | 卡片容器 | 统一 p-4 + rounded-lg |
| Badge | 标签/状态 | 2 种尺寸: default/sm |
| Table | 数据列表 | 候选人/职位列表的主展示方式 |
| Tabs | 标签页 | 候选人详情页的分区块切换 |
| Dropdown Menu | 下拉菜单 | 更多操作、批量操作 |
| Command | 命令面板 | 全局搜索 (Cmd+K) |
| Toast | 消息提示 | 操作反馈（成功/失败） |
| Skeleton | 加载骨架 | 数据加载时的占位展示 |
| Tooltip | 提示文字 | 引用标记悬停、图标说明 |
| Popover | 弹出面板 | 日期选择、筛选面板 |
| Avatar | 头像 | 候选人头像（有图用图，无图用首字母） |
| Separator | 分割线 | 区块分隔 |
| Collapsible | 折叠面板 | 画像区块的渐进披露 |

### 6.2 组件定制规则

1. shadcn/ui 组件代码拷贝进项目后，允许修改样式但不得改变 API（props）
2. 如需扩展组件功能，通过 wrapper 组件实现，不直接修改 ui/ 下的文件
3. 业务组件放在 `components/candidates/`、`components/jobs/`、`components/ai/` 等目录
4. 所有组件必须支持深色主题，通过 CSS Variables 自动切换

### 6.3 按钮 使用规范

| Variant | 场景 | 视觉 |
|---------|------|------|
| default (filled) | 主操作：保存、确认、提交 | 品牌蓝背景 + 白色文字 |
| secondary | 次要操作：取消、重置 | 浅灰背景 + 深色文字 |
| outline | 辅助操作：筛选、导出 | 透明背景 + 边框 |
| ghost | 最低频操作：更多、折叠 | 透明背景 + 无边框 |
| destructive | 危险操作：删除、移除 | 红色背景 + 白色文字 |

按钮尺寸：
| Size | 高度 | 用途 |
|------|------|------|
| sm | h-8 (32px) | 表格内操作、紧凑场景 |
| default | h-9 (36px) | 表单、对话框 |
| lg | h-10 (40px) | 页面主操作 |

**约束：**
- 每个区块最多 1 个 primary 按钮
- 危险操作必须与主操作间距 >= 8px
- 按钮内文字不超过 4 个汉字（中文场景）

## 7. AI 功能视觉标记系统

这是 HireMind 设计系统的核心差异化部分。所有 AI 生成或推理的内容必须有清晰的视觉标记，让用户一眼区分事实数据与 AI 推理。

### 7.1 推理类型视觉映射

基于 TAD 中 Prisma Schema 的 InferenceType 枚举，两种基础类型有明确的视觉区分：

#### FACT_EXTRACTION（事实提取）

从简历中直接提取的结构化数据。

**视觉规则：**
- 背景色：`var(--ai-fact-muted)` — 浅蓝背景
- 左侧竖线：3px 宽，`var(--ai-fact)` 蓝色
- 标记图标：Lucide `FileText` (16px)
- 标记文字："提取自简历" / "来源：XX简历"
- 引用标记：悬停显示原文摘录 (sourceText)，点击跳转简历详情

```
┌─ 左侧蓝色竖线 3px ──────────────────────────────────┐
│  📄 [提取自简历]  张三，5年前端开发经验               │
│  [悬停] → 显示原文："2019年至今，担任前端开发工程师..."  │
│  [点击] → 跳转到简历原文对应位置                       │
└──────────────────────────────────────────────────────┘
```

#### IMPLICIT_REASONING（隐性推理）

AI 基于数据推断的结论，非原文直接陈述。

**视觉规则：**
- 背景色：`var(--ai-reasoning-muted)` — 浅紫背景
- 左侧竖线：3px 宽，`var(--ai-reasoning)` 紫色
- 标记图标：Lucide `Sparkles` (16px)
- 标记文字："AI 推理" / "AI 分析"
- 置信度展示：置信度数值 + 进度条（confidence 字段）
- 依据链展示：点击展开推理依据列表 (reasoningChain)
- 渐变边框效果：使用 AI 渐变色的微弱渐变边框

```
┌─ 左侧紫色竖线 3px ──────────────────────────────────┐
│  ✨ [AI 推理 · 置信度 85%]                            │
│  该候选人的技术栈偏向前端，后端经验较少，                │
│  建议 衡量其全栈能力是否满足岗位需求。                  │
│                                                       │
│  [推理依据 ▼] ← 点击展开                              │
│  ├─ 依据1：简历中前端项目占比 80%                      │
│  └─ 依据2：最近 3 份工作均为前端职位                   │
│                                                       │
│  [AI 渐变微弱边框: violet→indigo]                      │
└──────────────────────────────────────────────────────┘
```

#### 其他推理类型视觉映射

其他 InferenceType 值使用统一 AI 样式，但 Badge 文字不同：

| InferenceType | Badge 文字 | 图标 |
|---------------|-----------|------|
| SKILL_ASSESSMENT | 技能评估 | `Award` |
| EXPERIENCE_ANALYSIS | 经历分析 | `TrendingUp` |
| CULTURE_FIT | 文化契合 | `Users` |
| CAREER_TRAJECTORY | 职业轨迹 | `Route` |
| RISK_ASSESSMENT | 风险评估 | `AlertTriangle` |

### 7.2 论文式引用标记

每条 AI 画像信息旁显示引用标记，遵循学术论文引用标注风格。

**引用标记样式：**

```
格式：[n]  其中 n 为引用编号
样式：text-xs，上标，var(--ai-primary) 颜色
交互：悬停显示出处摘要 Tooltip，点击跳转原文
```

```
候选人拥有 5 年前端开发经验 [1]，
擅长 React 和 TypeScript 生态 [2]，
具有技术团队管理经验 [AI推理·3]。
                ↑             ↑           ↑
           事实提取         事实提取    隐性推理（不同样式）

悬停 [1] → Tooltip: "来源：张三_简历_2024.pdf - 第2段"
悬停 [2] → Tooltip: "来源：张三_简历_2024.pdf - 技能清单"
悬停 [AI推理·3] → Tooltip: "AI 基于项目规模和团队构成推断，置信度 78%"
```

**引用编号颜色区分：**
- 事实引用 `[n]`：蓝色 (`var(--ai-fact)`)，背景浅蓝
- AI 推理引用 `[AI推理·n]`：紫色 (`var(--ai-reasoning)`)，背景浅紫

### 7.3 AI 画像卡片

AI 画像在候选人详情页以卡片形式呈现，遵循 360 度全息视图的分区块布局。

**画像卡片结构：**

```

├──────────────────────────────────────────────┤
│                                               │
│  ┌─ 摘要 ──────────────────────────────────┐  │
│  │  张三是资深前端工程师，5年经验，           │  │
│  │  擅长 React 生态，有团队管理经验。[AI推理] │  │
│  └──────────────────────────────────────────┘  │
│                                               │

│                                               │
└──────────────────────────────────────────────┘
```

**画像卡片视觉规范：**

- 卡片边框：1px `var(--border)` + 微弱 AI 渐变阴影 (`var(--shadow-ai)`)
- 卡片顶部：AI 图标 + "AI 画像" 文字 + 生成时间 + 操作按钮
- 区块标题：H4 字号，font-medium，带折叠箭头
- 区块内容：Body 字号，引用标记内联
- 默认展开：摘要、优势、风险点
- 默认折叠：技能分析、工作经历、教育背景
- 每个区块右上角：区块级 AI 标记（如 "AI 分析 · 3 条推理"）

### 7.4 AI 功能的零侵入交互规范

AI 功能不能打断用户的核心工作流。

**允许的 AI 交互方式：**
- 折叠面板内的 AI 建议区块
- 列表项旁的 AI 标记 Badge
- 卡片内的 AI 推理区块
- 悬停触发的 AI 分析 Tooltip
- 页面右下角的 AI 助手入口（不遮挡内容）

**禁止的 AI 交互方式：**
- 弹窗 / 模态框 展示 AI 结果
- 自动跳转到 AI 功能页面
- AI 内容遮挡核心数据
- 闪烁/动画吸引注意力到 AI 内容
- 全屏 AI 面板

**降级状态：**
- AI 服务不可用时：AI 区域显示 "AI 分析暂不可用" + Skeleton 占位
- AI 正在生成时：Skeleton + 脉冲动画，不阻塞页面其他内容
- AI 生成失败时：显示 "生成失败" + 重试按钮，不影响页面使用

### 7.5 AI 状态视觉规范

| 状态 | 视觉 | 说明 |
|------|------|------|
| 未生成 | 空状态插图 + "生成 AI 画像" 按钮 | 首次进入候选人详情 |
| 生成中 | Skeleton 骨架屏 + 脉冲动画 | 不阻塞页面 |
| 已生成 | 完整画像卡片，区块折叠/展开 | 默认状态 |
| 生成失败 | 错误提示 + "重试" 按钮 | 不影响其他内容 |
| 已过期 | 提示 "简历已更新，画像需刷新" + 刷新按钮 | 简历变更后 |
| 服务降级 | "AI 分析暂不可用" + Skeleton | AI 服务不可用 |

## 10. 交互状态规范

### 10.1 通用交互状态

| 状态 | 视觉 | 说明 |
|------|------|------|
| Default | 基础样式 | 正常展示 |
| Hover | 背景色变深、鼠标变 pointer | 悬停 |
| Active/Pressed | 背景色更深、轻微缩小 scale(0.98) | 按下 |
| Focused | ring-2 ring-ring 聚焦环 | 键盘聚焦 |
| Disabled | opacity-50 + cursor-not-allowed | 不可操作 |
| Loading | Spinner 或 Skeleton | 数据加载中 |
| Empty | 插图 + 引导文字 + 操作按钮 | 无数据 |
| Error | 红色文字/图标 + 错误描述 + 重试 | 出错 |

### 10.2 加载状态

- 列表页：表格行 Skeleton（3-5 行）
- 详情页：区块级 Skeleton
- AI 画像：完整卡片 Skeleton + 脉冲动画
- 按钮：内置 Spinner，文字变为 "处理中..."
- 不使用全屏 Loading 遮罩

### 10.3 空状态

每个列表和区块必须有空状态设计：
- 简洁的 SVG 插图（不超过 120x120px）
- 说明文字："暂无 XX"
- 引导操作按钮："新建第一个 XX"
- 不使用大段文字说明

## 11. 可访问性规范

### 11.1 色彩对比度

- 正文文字与背景对比度 >= 4.5:1 (WCAG AA)
- 大文字 (18px+) 与背景对比度 >= 3:1
- 交互元素与背景对比度 >= 3:1
- 不依赖颜色传达信息（需配合图标或文字）

### 11.2 键盘操作

- 所有交互元素可通过 Tab 键访问
- 焦点顺序从上到下、从左到右
- 焦点状态有明显视觉标记 (ring-2)
- Esc 关闭弹窗/下拉菜单
- Enter 触发按钮/提交表单

### 11.3 屏幕阅读器

- 所有图片有 alt 文字
- 图标按钮有 aria-label
- 表单输入有关联 label
- 动态内容更新有 aria-live 区域
- AI 内容有 aria-label 标注类型（"AI 推理"/"事实提取"）

## 12. 动效规范

### 12.1 动效原则

- 动效服务于信息反馈，不为装饰
- 时长 150-300ms，不超过 500ms
- 缓动函数：ease-in-out
- 尊重 prefers-reduced-motion 系统设置

### 12.2 标准动效

| 场景 | 时长 | 效果 |
|------|------|------|
| 悬停反馈 | 150ms | 背景色 transition |
| 下拉展开 | 200ms | 高度 + opacity |
| 页面切换 | 200ms | opacity fade |
| Toast 弹出 | 300ms | 从底部滑入 |
| 折叠展开 | 200ms | 高度变化 |
| AI 画像生成 | 1000ms | Skeleton 脉冲 |

### 12.3 AI 专属动效

- AI 推理区块出现时：从上方轻微滑入 (translateY 8px -> 0) + opacity fade，300ms
- 引用标记悬停：Tooltip 展开 200ms
- AI 生成完成：区块替换 Skeleton 时有轻微 scale 弹跳 (scale 0.98 -> 1.02 -> 1)，400ms
- 不使用持续闪烁/脉冲效果（分散注意力）
