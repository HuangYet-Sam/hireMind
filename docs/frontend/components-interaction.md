## 1. 全局交互规范（§10.1a）

### 🤖AI操作菜单（每个页面右上角）

**职责边界**：AI操作菜单 = 主动执行（批量操作、一键触发、任务委托）；AI Context Bar = 被动洞察。

| 页面 | AI操作菜单项 |
|------|------------|
| 岗位管理 | AI批量生成JD / AI分析空缺风险 / AI推荐关闭过期岗位 |
| 候选人管理 | AI批量推荐评语 / AI扫描沉睡候选人 |
| 面试管理 | AI批量生成考察清单 / AI分析面试通过率 |
| Offer管理 | AI生成薪资建议 / AI分析Offer接受率 |
| 招聘分析 | AI生成洞察报告 / AI预测招聘周期 |

### 双模创建入口规范

所有"创建"类操作统一提供双模入口，切换按钮位于表单顶部。

| 适用范围 | 功能模块 | 说明 |
|----------|---------|------|
| ✅ 双模(AI引导+手动) | 岗位创建 | 🤖AI引导对话式 + 📝手动表单 |
| ✅ 双模(AI引导+手动) | 面试创建 | 🤖AI引导对话式 + 📝手动表单 |
| 🔵 仅AI预填 | Offer创建 | AI预填方案 + 手动编辑 |
| 🔵 仅AI预填 | 候选人创建 | 手动创建（AI辅助简历解析） |
| ❌ 不支持 | 简历上传 | 纯拖拽上传 |
| ❌ 不支持 | 反馈表单 | 纯表单填写 |

### AI交互决策矩阵

| 用户意图 | 入口 | 结果展示 |
|---------|------|---------|
| 快速AI操作 | 🤖页面右上角AI操作菜单 | Toast+原地刷新列表 |
| 深度AI对话 | Agent Sidecar（右侧抽屉） | Sidecar内SSE流式展示 |
| 自由对话 | 全屏Chat（/chat） | Chat页面内流式展示 |

**路由规则**：AI操作菜单完成后，Toast提供「在Sidecar中继续」链接。

## 2. 页面状态设计规范（§10.5）

| 状态类型 | 触发条件 | UI规范 | 交互 |
|---------|---------|--------|------|
| **空状态** | 首次加载无数据 | 插画+引导文案+CTA按钮 | CTA触发创建流程 |
| **加载态** | API请求中 | 骨架屏（Skeleton），非Spinner | 不阻塞导航 |
| **错误态** | API失败/超时 | 错误图标+描述+重试按钮 | 重试重新请求 |
| **部分数据** | 有数据但不完整 | 已有数据正常展示，缺省字段"待补充" | 点击可编辑 |
| **权限不足** | 无访问权限 | 锁定图标+"您无权访问"+申请权限 | 触发权限申请 |
| **离线态** | 网络断开 | 顶部黄色Banner | 自动重连后消失 |

**实现约定**：所有状态由Pinia store统一状态机管理，API返回`{status, data, error}`标准结构。

## 3. AI引导式创建交互流程（§10.9）

### 5步交互

| Step | 交互 | UI | 降级 |
|------|------|-----|------|
| 1. 输入 | 自然语言描述 | 页面内嵌输入框(非Sidecar) | — |
| 2. 解析 | AI解析→结构化预览(🟢高/🟡中/🔴低置信度) | 右侧实时预览卡片 | 失败→"无法理解，请手动填写" |
| 3. 确认 | AI逐项确认关键参数 | 对话式逐项确认 | — |
| 4. 预览 | 最终表单预览，AI字段蓝色高亮 | 完整表单(可编辑) | — |
| 5. 创建 | 确认创建，后台执行 | Toast通知结果 | — |

### 降级矩阵

| 异常场景 | UX降级方案 | 数据保护 |
|---------|-----------|---------|
| AI解析超时(>15s) | 骨架屏+倒计时+"⏹️取消"，超时后建议切换手动 | 原始输入保留 |
| 低置信度(<0.6) | 橙色高亮+⚠️+AI建议值作placeholder | 不自动提交 |
| AI完全解析失败 | 友好错误+原始文本复制到剪贴板+自动切手动 | 输入不丢失 |
| 模式中途切换 | **自动映射已解析字段到手动表单** | 已解析数据完整保留 |
| AI部分解析成功 | 成功字段蓝色高亮，失败标红+"请手动填写" | 成功字段保留 |
| 网络中断 | "网络异常"+"重试"/"切换手动"，本地暂存 | 全部本地暂存 |

## 4. 面试视图切换（§10.10）

| 视图 | 适用场景 | 切换方式 | 默认 |
|------|---------|---------|:----:|
| 📅日历视图 | 查看本周/月面试排期 | 顶部Tab切换 | ✅ |
| 📋看板视图 | 按状态管理 | 顶部Tab切换 | |
| 📄列表视图 | 批量操作/筛选/导出 | 顶部Tab切换 | |

### 视图切换状态同步

- **筛选条件保持**：视图切换后保留筛选条件
- **选中项保持**：跨视图选中高亮
- **分页/滚动不保持**：视图切换重置滚动
- **新建/编辑后**：自动刷新当前视图数据，回到操作前视图
- **视图持久化**：最近选择存入`localStorage`

## 5. 外部Token URL页面（§10.10a）

面向流程参与者，独立于HR主系统，无侧边栏/顶部导航。

### 面试反馈页 `/feedback/:token`

- **布局**：移动端优先
- **组件**：候选人信息卡片(只读) + 评分区(1-5星，≥44px) + 自由反馈文本框 + 提交按钮(全宽≥48px)
- **状态机**：Token验证中 → Token有效 → 填写反馈 → 提交中 → 提交成功
- **异常态**：Token过期/已使用 → "该链接已失效，请联系HR"

### 入职自助页 `/onboarding/:token`

- **布局**：移动端优先
- **组件**：入职信息展示 + 材料清单(逐项上传) + 文件上传(相机直拍) + 进度条
- **状态机**：Token验证中 → 材料待提交 → 审核中 → 审核通过 → 入职完成
- **文件限制**：单文件≤10MB

### 共享招聘看板 `/share/:token/dashboard`

- **布局**：桌面端优先，只读
- **内容**：精简版招聘分析（仅关联岗位漏斗+关键指标）
- **隐藏**：AI操作菜单、创建入口、编辑按钮
- **脱敏**：薪资范围隐藏、候选人联系方式隐藏
- **底部水印**："由HireMind提供"

### 移动端适配规范

| 维度 | 规范 |
|------|------|
| 响应式断点 | <768px单列 / ≥768px双列 |
| 评分组件 | 星星≥44px，触摸友好 |
| 表单输入 | 全宽，大按钮≥48px |
| 状态提示 | 全屏Toast替代桌面右上角 |
| 安全 | Token 7天有效，一次性使用，10次/分钟限流 |

### 候选人外部页面（Phase 2）

| 路由 | 功能 |
|------|------|
| `/candidate/:token/profile` | 候选人自助更新信息 |
| `/candidate/:token/offer` | 候选人查看Offer并回应 |
